### 常见面试问：怎么定位并解决性能瓶颈？
- **定位顺序**：Profiler/Timeline 确认 CPU 或 GPU 瓶颈 → CPU 查看 Scripts/GC/Physics/Animation，GPU 查看 Batches/SetPass/Overdraw → 使用 Memory Profiler 查大资源/泄漏。
- **复现原则**：真机、Release、接近上线资源；录制 Profiler 数据包并附带场景操作步骤。

### GC/内存优化的三板斧
- 避免频繁分配：对象池复用、减少 `string` 拼接（用 `StringBuilder`）、避免 `LINQ` 生成闭包和临时列表。
- 减少装箱：使用泛型集合与 `struct` 迭代器；日志/Debug 关闭或用 `Conditional`。
- 资源生命周期：Addressables 使用 `Release`/`ReleaseInstance`，场景切换前显式卸载无用依赖；大对象加载改为异步，避免主线程卡顿。

### Draw Call/渲染优化常考点
- 批处理：开启 SRP Batcher/GPUI，合并材质/贴图（Atlas），减少材质实例化；实例化渲染用 `Graphics.DrawMeshInstanced` / DOTS Hybrid。
- Overdraw：UI 分层动静分离，使用 `Mask` 时谨慎；3D 物体按距离排序裁剪，使用 LOD/Occlusion Culling。
- 后处理与阴影：移动端谨慎使用全屏后处理，阴影采用 Blob/烘焙，实时阴影减分辨率与距离。
